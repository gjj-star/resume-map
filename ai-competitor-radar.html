AI竞品雷达 · 产品设计说明

高保真交互原型（本页随原型一同交付）。下文为 PRD 摘要，用于说明产品定位、假设与迭代路线。

1产品定位

一句话
面向产品/战略/投资人群的 AI 竞品分析助手：输入任意赛道，秒级生成结构化竞品对比与动态情报。
价值主张
把"人工搜集 + 排版 2 天"的竞品分析，压缩为"输入即出报告"的 2 分钟动作，且输出格式统一、可复用。
当前形态
高保真网页原型，数据为 Mock。用于验证信息架构与交互，未接真实模型。

2目标用户与场景

  • 产品经理：新需求立项前快速摸底赛道格局，做汇报素材。
  • 战略/商业分析：周期性跟踪竞品动作，输出监测简报。
  • 投资人/研究员：初筛赛道时快速建立对比框架。
  • 典型场景：老板丢来"看看 AI 笔记现在谁家强" → 打开雷达输入 → 拿报告进会。

3核心功能

  • 智能维度生成：根据赛道自动推荐合适的对比维度（而非固定模板）。
  • 竞品矩阵：卡片化呈现各家定位、亮点、定价与一句点评。
  • 功能对比表：可勾选的功能矩阵，支持强弱可视化。
  • 动态情报：近期行业/产品动作时间线，带日期与摘要。
  • 导出：一键生成 PDF / 分享链接，便于进会汇报。

4假设的 AI 能力（接真实模型时）

模型
LLM（如 DeepSeek / 通义 / GPT 类），后端代理调用，前端不暴露密钥。
Prompt 策略
赛道分类 → 维度模板召回 → 竞品召回（RAG/检索）→ 结构化 JSON 输出（固定 schema 保证前端可渲染）。
数据来源
公开资讯/官网/应用商店评论；动态情报接新闻/社媒检索 API。
可靠性
输出须带"数据来源/时间"标签，关键结论可溯源，避免幻觉。

5信息架构

输入层
首页搜索框 + 热门赛道快捷入口 + 历史记录。
处理层
加载态（扫描动画）→ 结构化生成。
结果层
洞察 → 维度 → 竞品矩阵 → 对比表 → 动态 → 导出/重生成。
详情层
竞品卡片点击 → 弹窗展示深度 AI 点评。

6迭代路线

V0 当前
高保真原型
验证交互与信息架构，Mock 数据。
V1
接真实模型
接入 LLM + 检索，输出可溯源。
V2
监测与协作
订阅赛道、周报推送、团队共享看板。
V3
深度洞察
趋势预测、SWOT 自动生成、导出 PPT。

7数据说明

本原型中所有竞品信息、定价、动态均为演示用 Mock 数据，不代表真实情况。接真实模型后，数据应由检索结果实时生成并标注来源。
